Section 1

背景与目标

为什么需要通用的 Action Card

Tanka 在小助手和 Flow 中都需要越来越强的执行能力。目前已对接 50+ 外部 MCP 和各类 Tool，加上 Tanka 自建功能，共计 60+ 可执行 Action。逐个为每个 Tool 设计定制化表单不可维护、不可扩展。

参考 Claude Code 的通用审核模式：AI 在执行任何外部操作前，先展示操作内容让用户确认。我们在此基础上增加了两层编辑的设计 — 默认的 Action Card 详情页以统一的 Markdown 格式展示 AI 即将执行的操作（适配所有第三方 Tool）；对于 Tanka 自建功能，用户还可以点击 "Edit Manually" 拉起原生表单进行精细编辑。这样既保持了架构的通用性，又不浪费 Tanka 已有的功能。

适用范围

Action Card 同时覆盖 AI 小助手 和 Flow 两个场景的 Action 能力。Permission 机制两边通用。

Phase I 交付物

交付物说明
TriggerAI 检测信号 → 对话气泡通知（内联半透明胶囊）
Action Card统一卡片组件，3 种语义：Connect / Permission / Vote

核心原则

  • AI 执行任何外部操作前，必须通过 Action Card 让用户审核确认
  • 一套 Action Card + 一套 Markdown 详情覆盖所有 60+ Tool，无需逐个设计表单
  • Tanka 自建功能额外提供 Edit Manually → 原生表单，精细编辑后发送
  • 同一时刻只有一张 waiting 卡片，逐张推送
  • 用户可随时跳过（直接发消息），交互不阻塞
  • 设计哲学：通过通用的 AI native 审核交互养成用户习惯，逐步建立信任，最终用户不再需要逐步审核

Section 4

全界面功能规格总表

每行 = 一个独立界面状态。功能描述列包含完整的 UI 元素、交互细节、数据来源和边界条件。

Trigger

T-1 · Trigger 气泡

功能描述页面（原型图）

AI 助手面板对话流中，作为普通 AI 消息气泡展示。气泡内容包含：

  • 来源胶囊：内联半透明胶囊（rgba(0,150,248,0.12) 背景 + 群聊图标 + 群名，如 [🔵 AI Lab]），与文字同行排列
  • 摘要文案：个人化口吻，如"刚刚在讨论开会的事，大家提到了会议安排：'明天两点开个会讨论 Q2 计划'"
  • 参会人信息："David, Sarah, Mike 已确认参会"
  • 行动建议：primary 色文字，如"可能和你关注的事情相关，要不要我帮你安排？"
  • 发起人：气泡底部左侧显示发起人头像（18px 圆形）+ 名字（11px 灰色）

交互细节：

  • 来源胶囊可点击，跳转到对应群聊的原始消息位置
  • Push 通知复用 AI 助手消息的系统 push，无需单独实现

数据来源：

  • signal.type = "meeting_intent"，confidence ≥ 0.7 时触发
  • source.channel_id 用于渲染胶囊中的群聊名
  • extracted_entities（时间、参会人、主题）用于生成摘要文案

群聊侧表现：

  • 群聊对话流中不显示任何信号卡片或通知条
  • AI 在后台静默记录，仅在小助手面板通知用户

C-2 · OAuth · loading

功能描述页面（原型图）

用户点击 App 选项（如 Google Calendar）后，弹出 OAuth 授权弹窗。

C-4 · OAuth · failed

功能描述页面（原型图）

OAuth 回调失败或用户主动取消授权后，弹窗内容变更：

  • Provider Logo 替换为红色 ✘ 图标（52px 圆形，background: #ef4444）
  • 文案变为"连接失败"（红色粗体）+ "请稍后重试"

持续时间：~1s 后自动关闭弹窗

后续逻辑（含重试计数）：

  • 第 1 次失败：弹窗关闭后，卡片保持 waiting。用户之前点击的 App 选项变为 retry 按钮样式（可再次点击）。AI 发消息提示："连接失败，请重试。"
  • 第 2 次失败：AI 提供 fallback 方案："连接暂时不成功，你也可以稍后在设置中连接。我先继续帮你推进其他步骤。" → AI 继续推进不依赖该连接的流程，不再阻塞在此步骤
  • retry 无次数上限，用户可继续点击。但 AI 不再重复发失败提示

Section 6

数据模型

TankaActionCard

interface TankaActionCard {
  id: string
  mode: 'vote'
  title: string
  body: string                          // Markdown（支持表格）

  options: Array<{
    id: string
    label: string                       // 仅标题，无副标题
    emoji: '👍' | '👎'
  }>

  // 即将执行的 Action（详情页顶部 Action Entry）
  action?: {
    tool: string                        // "Zoom" | "HubSpot" | "Google Calendar"
    api: string                         // "create_meeting" | "update_contact_email"
    builtin: boolean                    // true = Tanka 内建（有 Edit Manually）, false = 外部 MCP
    prefill?: Record<string, any>       // AI prefill 数据，传入内建表单
  }

  permission?: {                        // 有此字段 → Permission 语义
    tool: string                        // MCP 返回，如 "Google Calendar"
    level: 'read' | 'write' | 'delete'
  }
  relatedPermissions?: string[]         // 有此字段 → Vote 语义

  status: 'waiting' | 'answered' | 'skipped'
  createdAt: Date
}
